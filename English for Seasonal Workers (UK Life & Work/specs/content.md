# 📝 Content Development — FarmEnglish A0-A1

## 🏆 Золотой стандарт (Zero-level pedagogical consistency)

Основной принцип разработки уроков — **строгая лексическая изоляция и преемственность**. 

1. **Правило "Снежного кома":** В диалогах, примерах грамматики и тестах Урока N могут использоваться **только** слова из словаря текущего Урока N, а также словари из предыдущих уроков (от 1 до N-1).
2. **Никаких "случайных" слов:** Если слово (например, "apple" или "washroom attendant") не было явно введено в секции `words` текущего или предыдущих уроков — его использование **категорически запрещено**.
3. **Локализация инструкций:** Все задания и тесты должны быть полностью переведены (никаких английских "Fill the gap", только теги `[COMPLETE]`, `[TRANSLATE]`).
4. **Визуальная опора:** Обязательное использование кириллической транскрипции (`<span class="g-transcr">`) для каждого английского слова в правилах и таблицах.

### 🛑 СТРОГИЙ ЧЕК-ЛИСТ (Автоматически проверяется скриптами)
> **ВАЖНО:** Эти правила не просто текст. Они жестко запрограммированы в наших скриптах тестирования (`scratch/audit.js` и `scratch/advanced_audit.js`). Любое нарушение приведет к блокировке релиза урока!
При проверке и создании КАЖДОГО урока (до и после) должны выполняться следующие **непреложные условия**:
1. **ПОЛНАЯ ЛОКАЛИЗАЦИЯ (CRITICAL):** Любой текстовый элемент UI, включая `intro`, `title`, объяснения в `quiz`, `cultural`, `note` и т.д., **ОБЯЗАН** присутствовать на ВСЕХ 5 языках: `_ru`, `_uz`, `_tj`, `_kg`, `_kz`. Пропуск хотя бы одного языка КАТЕГОРИЧЕСКИ ЗАПРЕЩЁН.
2. **Грамматика (Intro):** Блок `grammar` **обязан** содержать `intro` (на 5 языках) с наглядным примером (❌ Неправильно / ✅ Правильно).
3. **Грамматика (Примеры):** Суммарное количество примеров в таблицах `positive`, `negative` и `question` должно составлять **МИНИМУМ 10 штук**. Если тема требует, можно больше (11, 12...), но МЕНЬШЕ 10 категорически запрещено!
4. **Грамматика (Сетка правил):** Внутри `rule_ru`/`rule_uz`/и т.д. при перечислении списка слов или местоимений **обязательно** использование структурированной CSS-сетки: `<div style="display:grid;grid-template-columns:1fr 1fr;gap:4px">...</div>`. Текст сплошной строкой запрещён!
5. **Словарь (Количество):** Массив `words` должен содержать **МИНИМУМ 30 уникальных объектов**. Больше — можно, меньше — нельзя.
6. **Словарь (Группировка запрещена):** Каждое слово — это отдельный элемент массива. Запрещено группировать (например: `en: 'one, two, three'`). Только по одному!
7. **Диалоги (Количество):** Массив `dialogue` должен содержать **МИНИМУМ 10 реплик** (чередование m/w).
8. **Тесты (Количество):** Массив `quiz` должен содержать **МИНИМУМ 10 вопросов**. Каждая подсказка (`hint`) должна быть на 5 языках!
9. **Правило Снежного Кома:** Использовать ТОЛЬКО слова из текущего урока и всех предыдущих.
10. **Уникальность словаря (Запрет дубликатов):** Одно и то же английское слово не должно появляться в массиве `words` более чем в одном уроке. Каждое слово изучается только один раз.

### Грамматика — 3 формы + таблица + 10 примеров

**Методика подачи:**

1. Объяснения на простом языке без лингвистических терминов
2. Предполагается нулевой уровень знаний
3. Каждое правило обязательно в **трёх формах**: ✅ Утверждение / ❌ Отрицание / ❓ Вопрос
4. Визуальные таблицы: Английский + Кириллическая транскрипция + Перевод
5. Культурные заметки (объяснение концепций, которых нет в родном языке)
6. **Абсолютный перевод**: Все базовые частицы и местоимения должны переводиться явно в тексте правила (`I am (я)`), так как предполагается нулевой уровень знаний.
7. **Табличное структурирование (CSS Grid)**: Блоки правил форматируются двухколоночной сеткой (`<div style="display:grid;grid-template-columns:1fr 1fr;gap:4px">`) для предотвращения некрасивых переносов текста на маленьких экранах. Сгруппированные местоимения (He/She/It) необходимо разделять на отдельные строки.

**Формат данных грамматического блока:**

```javascript
grammar: {
  // Заголовок на каждом языке
  title_ru: 'Глагол TO BE: am / is / are',
  title_uz: "TO BE fe'li: am / is / are",
  title_tj: 'Феъли TO BE: am / is / are',
  title_kg: 'TO BE этиши: am / is / are',
  title_kz: 'TO BE етістігі: am / is / are',

  // Введение (максимально короткое, визуальное объяснение)
  // UX Правило: Intro ≠ правило. Intro = "понял за 3 секунды". Показывать через ❌ и ✅.
  intro_ru: '<div style="line-height:1.6">В английском так говорить ошибка:<br>❌ <b>I Ahmad</b><br>Правильно:<br>✅ <b>I am Ahmad</b><br>👉 Всегда добавляйте <b>am / is / are</b>.</div>',
  intro_uz: '<div style="line-height:1.6">Ingliz tilida bunday deyish xato:<br>❌ <b>I Ahmad</b><br>To‘g‘risi:<br>✅ <b>I am Ahmad</b><br>👉 Har doim <b>am / is / are</b> qo‘shish kerak.</div>',

  // ТРИ ОБЯЗАТЕЛЬНЫХ ФОРМЫ

  // Поскольку аудитория имеет низкую грамотность (и часто никогда не учила английский),
  // каждое базовое английское слово внутри правил грамматики (в сетках) ДОЛЖНО 
  // сопровождаться кириллической транскрипцией с помощью тега <span class="g-transcr">.
  forms: {
    positive: {
      label_ru: '✅ Утверждение',
      label_uz: '✅ Tasdiq',
      rule_ru: '<div style="display:grid;grid-template-columns:1fr 1fr;gap:4px"><div><code>I am</code> <span class="g-transcr">[ай эм]</span> (я)</div><div><code>We are</code> <span class="g-transcr">[уи ар]</span> (мы)</div><div><code>You are</code> <span class="g-transcr">[ю ар]</span> (вы/ты)</div><div><code>They are</code> <span class="g-transcr">[зэй ар]</span> (они)</div><div><code>He is</code> <span class="g-transcr">[хи из]</span> (он)</div><div><code>She is</code> <span class="g-transcr">[ши из]</span> (она)</div></div>',
      rule_uz: '<div style="display:grid;grid-template-columns:1fr 1fr;gap:4px"><div><code>I am</code> <span class="g-transcr">[ай эм]</span> (men)</div><div><code>We are</code> <span class="g-transcr">[уи ар]</span> (biz)</div><div><code>You are</code> <span class="g-transcr">[ю ар]</span> (siz/sen)</div><div><code>They are</code> <span class="g-transcr">[зэй ар]</span> (ular)</div><div><code>He is</code> <span class="g-transcr">[хи из]</span> (u, erkak)</div><div><code>She is</code> <span class="g-transcr">[ши из]</span> (u, ayol)</div></div>',
      // Визуальная таблица
      table: [
        {
          subj: 'I',
          verb: 'am',
          example: 'I am a worker',
          transcr: 'Ай эм э вокер',
          tr_ru: 'Я — рабочий',
          tr_uz: 'Men ishchiman',
          tr_tj: 'Ман коргар ҳастам',
          tr_kg: 'Мен жумушчумун',
          tr_kz: 'Мен жұмысшымын'
        },
        // ...ещё строки таблицы
      ]
    },
    negative: {
      label_ru: '❌ Отрицание',
      label_uz: '❌ Inkor',
      rule_ru: 'Добавляем <b>not</b>:<br><code>I am not</code> (я не)<br><code>He is not</code> (он не)<br><code>We are not</code> (мы не)',
      table: [ /* аналогично */ ]
    },
    question: {
      label_ru: '❓ Вопрос',
      label_uz: '❓ Savol',
      rule_ru: 'Глагол выходит вперёд:<br><code>Am I?</code> (я?)<br><code>Is he?</code> (он?)<br><code>Are you?</code> (вы?)',
      table: [ /* аналогично */ ]
    }
  },

  // Культурная заметка (различия между языками)
  cultural_ru: 'В узбекском/таджикском нет глагола "быть" в настоящем времени. В английском он ОБЯЗАТЕЛЕН!',
  cultural_uz: "O'zbek tilida hozirgi zamonda 'bo'lmoq' fe'li ishlatilmaydi. Ingliz tilida u MAJBURIY!",

  // Подсказка-предупреждение
  note_ru: '⚠️ НЕ говорите "I is" или "He am" — это ошибка!',
  note_uz: "⚠️ 'I is' yoki 'He am' demang — bu xato!",

  // 10 примеров из контекста фермы
  examples: [
    {
      en: 'I am Ahmad. I am from Tajikistan.',
      transcr: 'Ай эм Ахмад. Ай эм фром Таджикистан.',
      ru: 'Я — Ахмад. Я из Таджикистана.',
      uz: 'Men Ahmad. Men Tojikistondan.',
      tj: 'Ман Аҳмад ҳастам. Ман аз Тоҷикистон.',
      kg: 'Мен Ахмадмын. Мен Тажикстандан.',
      kz: 'Мен Ахмадпын. Мен Тәжікстаннан.'
    },
    // ...ещё 9 примеров
  ]
}
```

**Правила таблиц:**

| Колонка | Описание | Пример |
|---------|----------|--------|
| `subj` | Подлежащее | `I`, `You/We/They`, `He/She/It` |
| `verb` | Глагольная форма | `am`, `are`, `is` |
| `example` | Полный пример на английском | `I am a worker` |
| `transcr` | Кириллическая транскрипция | `Ай эм э вокер` |
| `tr_*` | Перевод на каждый из 5 языков | `Я — рабочий` |

### Кириллическая транскрипция

**Обязательна для:**

- Всех грамматических примеров (`examples[].transcr`)
- Всех строк таблиц (`table[].transcr`)
- Всех 30 слов урока (`words[].transcr`)
- Всех реплик диалога (`dialogue[].transcr`)

**Правила транскрипции:**

| Английский | Кириллица | Пример |
|-----------|-----------|--------|  
| th (voiced) | з/д | `the` → `зэ` |
| th (voiceless) | с/т | `think` → `синк` |
| w | в/у | `worker` → `вокер` |
| r | р | `farm` → `фарм` |
| a (long) | а | `farm` → `фарм` |
| a (short) | э | `cat` → `кэт` |
| i (long) | и | `team` → `тим` |
| i (short) | и | `pick` → `пик` |
| oo | у | `boot` → `бут` |
| er | ер/э | `worker` → `вокер` |

### Словарь — 30 слов на урок

**Формат каждого слова:**

```javascript
{
  e: '🍓',                    // Эмодзи (визуальная подсказка)
  en: 'strawberry',           // Английское слово
  transcr: 'стробери',        // Кириллическая транскрипция
  ru: 'клубника',             // Русский
  uz: 'qulupnay',             // Узбекский
  tj: 'тӯтӣ',                // Таджикский
  kg: 'кулпунай',             // Кыргызский
  kz: 'құлпынай'              // Казахский
}
```

**Правила отбора слов и эмодзи:**

1. Эмодзи (`e`) должны строго соответствовать точному смыслу слова, а не просто широкой ассоциации (например: для "passport" использовать 🛂, а не ✈️; для "office" использовать 🏢, а не карту 🗺️).
2. Слова должны быть ИСКЛЮЧИТЕЛЬНО из реальной жизни сезонных рабочих на британских фермах (поля, пакхаусы, кэмпы). **Категорически запрещено** добавлять «городские», «офисные» или нерелевантные слова (примеры грубых ошибок: "printer", "computer", "washroom attendant", "apple").
3. Прогрессия от простого к сложному.
4. **СТРОГО:** Без повторения слов между уроками (запрещено дублировать слова из предыдущих уроков, например, повторять "phone number").
5. **СТРОГО:** Каждый урок покрывает ровно одну тематическую область. Никаких слов, выбивающихся из темы (например, добавление "printer" в урок про сбор ягод).
6. Обязательно включать «survival» фразы (SOS, здоровье).
7. **СТРОГО:** Диалоги, примеры грамматики и тесты должны состоять ИСКЛЮЧИТЕЛЬНО из слов текущего словаря и словарей предыдущих уроков. Нельзя использовать слова, которые студент еще не изучал.

**Распределение 30 слов по типам:**

- 10 существительных (предметы, места)
- 8 глаголов (действия)
- 5 прилагательных (описания)
- 4 фразы-выражения
- 3 числа / время / даты

### Диалоги — 10 реплик на урок

**Формат реплики:**

```javascript
{
  s: 'm',                     // Спикер: m/w/c/d
  en: 'Good morning!',        // Английский
  transcr: 'Гуд монинг!',    // Кириллическая транскрипция
  ru: 'Доброе утро!',        // Русский
  uz: 'Xayrli tong!',        // Узбекский
  tj: 'Субҳ бахайр!',        // Таджикский
  kg: 'Кутман эртең!',       // Кыргызский
  kz: 'Қайырлы таң!'         // Казахский
}
```

**Спикеры:**

| Код | Роль | Контекст |
|-----|------|----------|
| `m` | Manager / Supervisor | Инструкции, правила |
| `w` | Worker (рабочий) | Вопросы, ответы |
| `c` | Cashier / Shop | Покупки |
| `d` | Doctor / Nurse | Здоровье |

**Правила диалогов:**

1. Диалог — связный сценарий из реальной жизни
2. Чередование спикеров (m→w→m→w...)
3. Использование грамматики текущего урока
4. Использование слов из текущего урока
5. Длина реплики: 5–15 слов

### Тесты — 10 заданий на урок

**Формат задания:**

```javascript
{
  q: '[COMPLETE] "I ___ Ahmad."',     // Вопрос с тегом локализованной инструкции
  opts: ['am', 'is', 'are', 'be'],   // 4 варианта
  c: 0,                              // Индекс правильного (0-based)
  expl: 'С местоимением I (я) всегда используется am.', // Объяснение (feedback)
  hint_ru: 'Я Ахмад.',               // Перевод контекста вопроса на русский (опционально)
  hint_uz: 'Men Ahmad.'              // Перевод контекста вопроса на другие языки...
}
```

**Теги инструкций для тестов:**
- `[COMPLETE]` ➔ ✍️ Заполните пропуск / Bo'sh joyni to'ldiring
- `[TRANSLATE]` ➔ 🗣️ Переведите / Tarjima qiling
- `[NEGATIVE]` ➔ ❌ Найдите отрицание / Inkor shaklni toping
- `[CORRECT]` ➔ ✅ Какое предложение правильное? / Qaysi gap to'g'ri?
- `[QUESTION]` ➔ ❓ Какой вопрос подходит к ответу: / Javobga qaysi savol mos keladi:

**Распределение 10 заданий:**

| # | Тип | Сложность |
|---|-----|-----------|
| 1–3 | Выбрать правильное слово | Лёгкий |
| 4–6 | Заполнить пропуск в предложении | Средний |
| 7–8 | Перевести фразу | Средний |
| 9–10 | Составить из контекста | Сложный |

**Правила тестов:**

1. Все задания используют слова/грамматику текущего урока
2. Правильный ответ случайно распределён (не всегда первый)
3. Неправильные варианты должны быть правдоподобными
4. Вопросы привязаны к фермерскому контексту
5. Каждое задание содержит `expl` — педагогическое объяснение ответа
6. **СТРОГОЕ ПРАВИЛО:** В вопросах категорически запрещено использовать английские слова или инструкции (типа "Complete", "Translate"), которые студент еще не проходил. Все инструкции должны задаваться через теги `[TAG]` (для перевода на родной язык), а сам контекст вопроса должен сопровождаться переводом `hint_lang`.
