# FarmEnglish — i18n Rules

## Supported Languages

`ru`, `uz`, `tj`, `kg`, `kz` — **Russian is the master/reference locale** (lingua franca for all target users).

## File Locations

- `translations.js` — shared i18n for `index.html`. Single file, all locales in one `const TRANSLATIONS = { ru: {...}, uz: {...}, ... }` object.
- Lesson files (`a1.html` etc.) — use an inline `TR` object for full autonomy (no external requests needed).

## How It Works

```javascript
// translations.js — global helper
function t(key, ...args) {
  const lang = window._lang || 'ru';
  const dict = TRANSLATIONS[lang] || TRANSLATIONS['ru'];
  const val = dict[key] ?? TRANSLATIONS['ru'][key] ?? key;
  return typeof val === 'function' ? val(...args) : val;
}

// Usage
t('nav_home')           // → "Главная" (if lang=ru)
t('done_of', 5, 15)     // → "5 / 15 уроков пройдено"
```

- `window._lang` — current language code, initialized from `localStorage('fe_lang')` on page load
- `setLang(code)` — updates `window._lang` and persists to `localStorage('fe_lang')`
- Fallback: if a key is missing in the current language → falls back to `ru`

## Lesson File i18n (Inline TR Object)

Lesson files use a local `TR` object instead of `translations.js`:

```javascript
const TR = {
  ru: { grammar: 'Грамматика', words: 'Слова', ... },
  uz: { grammar: 'Grammatika', words: "So'zlar", ... },
  tj: { grammar: 'Грамматика', words: 'Калимаҳо', ... },
  kg: { grammar: 'Грамматика', words: 'Сөздөр', ... },
  kz: { grammar: 'Грамматика', words: 'Сөздер', ... },
};
```

This keeps lesson files fully self-contained — they work without `translations.js`.

## Lesson Data Fields

All lesson content fields must be provided in all 5 languages:

```javascript
// Grammar — language-suffixed keys
title_ru, title_uz, title_tj, title_kg  // (kz uses kg as fallback if needed)
rule_ru, rule_uz, rule_tj, rule_kg
note_ru, note_uz, note_tj, note_kg

// Words, dialogue, examples — object with all 5 keys
{ en: '...', ru: '...', uz: '...', tj: '...', kg: '...', kz: '...' }
```

## Adding a New Key

1. Add the key to **`ru` first** — it is the source of truth
2. Add the same key to **all other 4 locales** (`uz`, `tj`, `kg`, `kz`)
3. Never leave a key missing in any locale — missing keys fall back to `ru` silently

## Key Naming Conventions

- `snake_case` only
- Prefix by context: `level_`, `quiz_`, `cat_`, `nav_`, `lang_`
- Generic UI: no prefix — `start`, `back`, `next`, `close`, `loading`
- Never use generic keys like `text1`, `label`, `msg` — be descriptive

## Rules

- Never hardcode user-visible strings in JS or HTML — always use `t()` or `TR[lang][key]`
- Exception: developer-only `console.log` messages — English only, no translation needed
- Do not delete keys from `ru` without removing them from all other locales
- Quiz feedback messages (`quiz_no_desc`) are arrays of 30 motivational strings — maintain all 5 language versions in sync
- Never write to localStorage directly for language — use `setLang(code)`
