### 1.2 Что НЕ готово / сломано (подтверждено)

**1.2.1 Аудит A1 проверял не тот контент (КРИТИЧНО для процесса).**
Все три скрипта читают `scratch/lessons_data*.js`, а не HTML. Для A2/B1 снимки байт-идентичны HTML,
но `scratch/lessons_data.js` (May 30 13:45) **старше** `a1.html` (May 30 14:58): различаются **12 из 15
уроков**, 70/450 словарных слотов, квизы в 9 уроках, диалоги в 5; урок 11 — вообще другой урок
(HTML: «Транспорт и направления» vs scratch: «this/that/these/those»). Следствие: список дублей B1
частично фантомный — «дубли» `lost`, `saturday` существуют только против устаревшего снимка, а
реальные дубли `left`, `town` (против настоящего a1.html) аудит **не видит**.

**1.2.2 На живом сайте нет ничего, кроме A1.**
`git ls-tree origin/master`: `a2.html` и `b1.html` отсутствуют (untracked), `index.html` модифицирован
и не закоммичен, 1 коммит (9306ca2) не запушен. Рабочий, прошедший 15 уроков A1 на проде, увидит
разблокированную карточку A2 → клик → **404**. Весь готовый A2 физически не доставлен пользователю.

**1.2.3 B1 — половина курса и провал аудита.**
6 уроков вместо обещанных 12 (README.md:21, specs/product.md:39). Живой прогон: `audit.js b1` —
дубль `light` (урок 5 ↔ урок 2); `check_duplicates.js all` — **30 кросс-курсовых дублей** против
A1/A2; `advanced_audit.js b1` → `ADVANCED_AUDIT_FAIL 1`. UI: каркас MODS на 15 уроков
(b1.html:1139-1200) — модули 4–6 рендерятся **пустыми заголовками** (:6249 молча пропускает
несуществующие уроки); баннер завершения требует `completedCount === 15` (:6232) и **недостижим**;
хаб при этом покажет «6 / 6» (index.html:1078) — противоречивые сигналы.

**1.2.4 Обещание «работает без интернета» не выполнено.**
README.md:5,34. Нет `sw.js`, нет manifest (ls + grep serviceWorker по всем файлам = 0). Файлы по
~1 МБ живут только в HTTP-кэше браузера, который на дешёвом Android вытесняется. Хуже: a1.html
**сам блокирует озвучку офлайн** — `if (!navigator.onLine) { …return }` (a1.html:1846-1850), хотя
Android TTS работает офлайн со скачанным голосом; тост об этом — хардкод только по-русски (:1832).
В a2/b1 блокировки нет (`u.onerror` → локализованный тост, a2:13592, b1:6186) — поведение курсов
противоречит друг другу.

**1.2.5 Квизы проходятся тапом по первому варианту.**
Распределение правильного ответа `c`: A1 — c=0 в 46 %, A2 — 47 % (c=3 — ни разу), B1 — **63 %**
(c=3 — ни разу). `Math.random`/shuffle в HTML-файлах: **0 вхождений** — опции рендерятся в
фиксированном порядке. Плюс 8 вопросов с утечкой ответа в тексте вопроса (6 в A1, 2 в A2; пример:
a1 L10 q7 `What is 'garlic'? (чеснок)` — ответ «чеснок» напечатан в вопросе).

**1.2.6 Snowball-проверка — фикция.**
content.md:54 утверждает: «advanced_audit.js … parses all text and cross-references the cumulative
vocabulary». В коде `advanced_audit.js` (все 202 строки) такой проверки **нет**. Доказанный ущерб:
диалог B1 L1 содержит лексемы вне кумулятивного словаря всех 36 уроков: `strawberry`, `another`,
`immediately`, `protective` и др. — слово «клубника» не изучается **нигде** в курсе для сборщиков
клубники.

**1.2.7 Документированный аудит-гейт не способен поймать провал.**
Команда из доков `node scratch/audit.js && node scratch/check_duplicates.js && node scratch/advanced_audit.js`
без аргументов проверяет только A1 (по устаревшему снимку!), `check_duplicates` по умолчанию
исключает B1, а `audit.js` **вообще не выставляет exit code** (0 даже при ошибках — проверено живым
прогоном). Живой прогон цепочки: все три exit 0 и «✅» — при 66 известных дублях в B1 (реальное число; 31 — по устаревшему снимку, пересчитано задачей 0.4).

**1.2.8 Паритет фич между уровнями отсутствует.**
В a2/b1 нет: сертификата (обещан в tasks.md:79; единственный grep-хит «cert» в a2 — словарная
карточка `certificate` :10543), экрана прогресса, разговорника. Разговорник существует **только
внутри a1.html** — steering/structure.md:7,44 и tech.md:19 неверно говорят, что он в хабе; вход с
хаба замаскирован под счётчик слов (index.html:879 `href="a1.html#phrases"`).

**1.2.9 Рассинхрон механик прогресса.**
Streak: a2/b1 считают день по **UTC** (`toISOString`, a2:13960), a1 — по **локальной** дате
(a1:2377-2380); занятие в 23:00 BST пишет «вчерашнюю» дату → серия ломается ровно в типичный для ЦА
вечерний сценарий. fe_words: три конкурирующих писателя — a1 перезаписывает суммой только своих
уроков (a1:2419), a2/b1 инкрементируют (+30), index «лечит» через `max(stored, totalDone*30)`
(:996-999) — счётчик мигает разными числами между страницами.

**1.2.10 Прочее подтверждённое.**
- Онбординг-кнопка «Начать Урок 1» ведёт на `a1.html?lesson=1`, но a1.html параметр **не парсит**
  (единственное вхождение — :2426 `replaceState`); a2/b1 при этом `?lesson=N` поддерживают.
- Замок уровней обходится прямым URL: в a2/b1 нет ни одной проверки прогресса предыдущего уровня.
- KZ-баг в index.html:1010: `үчүн` (кыргызское) вместо `үшін` в казахском заголовке; 5 inline-карт
  строк дублируют translations.js вопреки правилу i18n.md:78.
- Диалог A1 урока 2: все 10 реплик **без поля `kz`** (fallback на русский).
- A11y: `* { outline: none }` (index:55), смена языка недоступна с клавиатуры, `user-scalable=no`,
  `speechSynthesis` как голый идентификатор → ReferenceError на старых WebView без API (b1:6181).
- Переводы: kz===kg дословно в ~16 % слов (включая русские слова «ставка», «трактор» как «перевод»);
  uz со смешанной кириллицей-латиницей (`дам olish kuni`, b1 L4); кыргызское «кырсык» в hint_ru (B1 L1).
- Лексика-мусор для добивки квоты 30: A2 L14 — `fiscal year, dawn, dusk, fortnight`; B1 L4 — бренд
  `Monzo` как словарное слово + композиты `card details, chat support, phone call`.
- Эмодзи A2: 122 уникальных на 450 слов (📋 ×45, 💰 ×35) — карточки визуально не различаются.
